# Getting started

**Tailwind Variants** combines the power of TailwindCSS with a first-class variant API.

## Setup TailwindCSS

**Tailwind Variants** requires you to have TailwindCSS installed in your project. If you haven't installed
TailwindCSS yet, you can follow the [TailwindCSS installation guide](https://tailwindcss.com/docs/installation).

## Installation

To use **Tailwind Variants** in your project, you can install it as a dependency:

<Tabs items={['npm', 'yarn', 'pnpm']}>
  <Tab>

    ```bash copy
    npm install tailwind-variants
    ```

  </Tab>
  <Tab>

    ```bash copy
    yarn add tailwind-variants
    ```

  </Tab>
  <Tab>

    ```bash copy
    pnpm add tailwind-variants
    ```

  </Tab>
</Tabs>

### Choose Your Build (v3+)

Starting from v3, Tailwind Variants offers two build options:

#### Option 1: Original Build (with tailwind-merge)

For automatic Tailwind CSS conflict resolution, install `tailwind-merge` as a dependency:

<Tabs items={['npm', 'yarn', 'pnpm']}>
  <Tab>

    ```bash copy
    npm install tailwind-merge
    ```

  </Tab>
  <Tab>

    ```bash copy
    yarn add tailwind-merge
    ```

  </Tab>
  <Tab>

    ```bash copy
    pnpm add tailwind-merge
    ```

  </Tab>
</Tabs>

Then import from the default entry point:

```ts
import { tv, createTV, cn, cnMerge, cx } from 'tailwind-variants';
```

#### Option 2: Lite Build (without tailwind-merge)

For a smaller bundle size (~80% smaller), use the lite build which doesn't include `tailwind-merge`:

```ts
import { tv, createTV, cn, cnMerge, cx } from 'tailwind-variants/lite';
```

> **Note**: The lite build doesn't support the `twMerge` and `twMergeConfig` options. Choose this if bundle size is critical and you don't need automatic conflict resolution. Both `cn`, `cnMerge`, and `cx` are available in both builds, with `cn` and `cnMerge` providing conflict resolution in the original build (v3.2.2+).

## Usage

<br />

<TVComponent
  as="button"
  size="sm"
  color="secondary"
  tv={{
    base: 'font-medium bg-blue-500 text-white rounded-full active:opacity-80',
    variants: {
      color: {
        primary: 'bg-blue-500 text-white',
        secondary: 'bg-purple-500 text-white'
      },
      size: {
        sm: 'text-sm',
        md: 'text-base',
        lg: 'px-4 py-3 text-lg'
      }
    },
    defaultVariants: {
      size: 'md',
      color: 'primary'
    },
    compoundVariants: [
      {
        size: ['sm', 'md'],
        class: 'px-3 py-1'
      }
    ]
  }}
>
  Click me
</TVComponent>

```js copy
import { tv } from 'tailwind-variants';

const button = tv({
  base: 'font-medium bg-blue-500 text-white rounded-full active:opacity-80',
  variants: {
    color: {
      primary: 'bg-blue-500 text-white',
      secondary: 'bg-purple-500 text-white'
    },
    size: {
      sm: 'text-sm',
      md: 'text-base',
      lg: 'px-4 py-3 text-lg'
    }
  },
  compoundVariants: [
    {
      size: ['sm', 'md'],
      class: 'px-3 py-1'
    }
  ],
  defaultVariants: {
    size: 'md',
    color: 'primary'
  }
});

return (
  <button className={button({ size: 'sm', color: 'secondary' })}>
    Click me
  </button>
);
```

## IntelliSense setup (optional)

To enable autocompletion for **Tailwind Variants** you can follow the instructions below.

<Tabs items={['VSCode', 'Neovim', 'IntelliJ']}>
  <Tab>
    If you are using **VSCode** and the [**TailwindCSS IntelliSense Extension**](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss), you can use either of the following configurations in your `settings.json` file.

    **Option 1: Using `classFunctions` (Recommended)**

    This is the simplest approach and directly specifies the `tv` function:

    ```json copy
    {
      "tailwindCSS.classFunctions": ["tv"]
    }
    ```

    **Option 2: Using `classRegex`**

    For more advanced regex matching patterns:

    ```json copy
    {
      "tailwindCSS.experimental.classRegex": [
        ["(`.*?`)", "(\".*?\")", "('.*?')"]
      ]
    }
    ```

  </Tab>
  <Tab>
    If you are using **Neovim**, install [**Tailwind CSS Language Server**](https://github.com/neovim/nvim-lspconfig/blob/master/doc/server_configurations.md#tailwindcss) and add the following to your configuration.

    ```lua copy
    require('lspconfig').tailwindcss.setup({
      settings = {
        tailwindCSS = {
          experimental = {
            classRegex = {
              { "(`.*?`)", "(\".*?\")", "('.*?')" }
            },
          },
        },
      },
    })
    ```

  </Tab>
  <Tab>
    If you are using **IntelliJ** and the [**Tailwind CSS Plugin**](https://plugins.jetbrains.com/plugin/15321-tailwind-css), go to **settings** -> **Languages & Frameworks** -> **Style Sheets** -> **Tailwind CSS** and add the following to the configuration.

     ```json copy
     "experimental": {
       "configFile": null,
       "classRegex": [
          ["(`.*?`)", "(\".*?\")", "('.*?')"]
       ]
     }
    ```

  </Tab>
</Tabs>

## Prettier plugin setup (optional)

If you are using [prettier-plugin-tailwindcss](https://github.com/tailwindlabs/prettier-plugin-tailwindcss)
to sort your class names, you can add `tv` to the list of functions that
should be sorted.

```js
module.exports = {
  plugins: [require('prettier-plugin-tailwindcss')],
  tailwindFunctions: ['tv']
};
```
